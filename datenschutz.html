4. Kuenstliche Intelligenz (KI-Funktionen)

Die App nutzt KI-gesteuetzte Funktionen auf Basis von Google Gemini (ein generatives KI-Modell von Google LLC). Die Verarbeitung erfolgt ueber unsere eigenen Firebase Cloud Functions, die als Vermittler zur Gemini API dienen. Daten werden dabei ausschliesslich auf deine aktive Anfrage hin uebermittelt.

a) Bild-zu-Rezept-Erkennung
Wenn du ein Foto zur Rezepterkennung verwendest, wird das komprimierte Bild an unsere Cloud Function uebertragen und von dort an die Google Gemini Vision API weitergeleitet. Gemini analysiert das Bild und gibt strukturierte Rezeptdaten zurueck (Titel, Zutaten, Schritte, Portionen, Zeiten). Das Bild wird nach der Verarbeitung nicht dauerhaft auf Gemini-Servern gespeichert.

b) Sprache-zu-Rezept
Bei der Spracheingabe wird die Sprache zunaechst lokal auf deinem Geraet transkribiert (on-device Speech-to-Text). Der transkribierte Text wird anschliessend an Google Gemini uebermittelt, um daraus strukturierte Zutaten und Zubereitungsschritte zu erstellen. Die Sprachaufnahme selbst verlaesst das Geraet nicht.

c) Automatische Naehrwertberechnung
Die von dir eingegebene Zutatenliste eines Rezepts wird an Google Gemini uebermittelt, um Naehrwerte (Kalorien, Eiweiss, Kohlenhydrate, Fett) zu schaetzen. Diese Werte sind automatisch generierte Schaetzungen und ohne Gewaehr.

Rechtsgrundlage: Art. 6 Abs. 1 lit. b DSGVO (Vertragserfuellung, da die KI-Funktion auf aktive Anfrage ausgefuehrt wird).

Google LLC ist insoweit als Auftragsverarbeiter taetig. Gemaess den Nutzungsbedingungen der Gemini API werden ueber die API uebermittelte Daten nicht fuer das Training von KI-Modellen verwendet.

Weitere Informationen zur Gemini API: https://ai.google.dev/gemini-api/terms
Datenschutz bei Google AI: https://policies.google.com/privacy

5. Rezept-Import aus Drittquellen

Die App bietet die Moeglichkeit, Rezepte aus externen Quellen zu importieren. Dabei werden ausschliesslich die extrahierten Rezeptdaten gespeichert – keine personenbezogenen Daten der Drittplattformen.

  • Webseiten-Import (URL): Die angegebene URL wird von unserer Cloud Function serverseitig aufgerufen. Seiteninhalt und strukturierte Rezeptdaten (JSON-LD, Microdata) werden extrahiert.
  • Social-Media-Import (Instagram, TikTok, Pinterest): Oeffentlich zugaengliche Beitraege werden ueber oeffentliche Schnittstellen oder oeffentlich einsehbare Inhalte abgerufen. Gespeichert werden nur die extrahierten Rezeptdaten.
  • Chefkoch: Die App nutzt die oeffentliche Rezept-API von chefkoch.de zum Abruf von Rezeptdaten.

Du bist dafuer verantwortlich, dass du zur Nutzung der importierten Inhalte berechtigt bist und keine Urheberrechte Dritter verletzt.
